## Задача 1: Алгоритм Push-Sum

Для решения первой задачи был выбран алгоритм **Push-Sum**. Это децентрализованный метод вычисления среднего арифметического значения всех узлов в сети.

**Описание алгоритма:**
Каждый агент хранит два значения: сумму s и вес w.

* На каждой итерации агент отправляет половину своих значений  случайному соседу, а вторую половину оставляет себе.
* Текущая оценка среднего значения вычисляется как отношение s/w.
* Алгоритм гарантирует сходимость к истинному среднему значению при условии связности графа.

**Как запустить:**

* Основная реализация c выводом данных в консоль:
```
python pushsum.py

```


* Реализация с наглядной анимацией:
```
python visualized_pushsum.py

```

## Задача 2: Local Voting Protocol (LVP)

Реализация второй задачи находится в файле **`lvp.py`**.

**Local Voting Protocol** (Протокол локального голосования) предназначен для достижения консенсуса в условиях "зашумленной" среды. Агенты корректируют свое состояние, ориентируясь на значения своих соседей, с учетом заданного шага (step size).

В данной реализации учтены:

1. **Шум**: добавление случайных колебаний к значениям.
2. **Задержки**: сообщения могут доходить до соседа не сразу, а через несколько итераций.
3. **Отказы**: агенты могут временно отключаться от сети.

**Как запустить:**

* Реализация c выводом данных в консоль:
```
python lvp.py

```

## Вспомогательные инструменты

### `utils.py`

Файл содержит общую логику для обеих задач:

* **Генерация графа**: функция `generate_random_connected_graph` создает случайную связную топологию сети.
* **Расчет стоимости**: содержит `cost_dictionary` (веса для памяти, сообщений и операций).

## Настройка симуляции

Вы можете свободно менять параметры симуляции в соответствующих `.py` файлах:

* **Количество агентов**: измените переменную `NUMBER_OF_AGENTS`.
* **Количество итераций**: измените параметр `max_rounds`.
* **Топология сети**: можно изменять плотность связей через параметр p в `generate_random_connected_graph`.
